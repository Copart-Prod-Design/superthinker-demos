# Superthinker Demos

Public, shareable prototype demos generated by [superthinker](https://github.com/Copart-Prod-Design/superthinker) — Copart's PM PRD-critique skill.

Each subfolder is one project's prototype, built on the [Vallejo design system](https://github.com/Copart-Prod-Design/Vallejo) and served live via GitHub Pages.

> **Public on purpose.** Only finished, shareable *prototypes* live here. The PRDs, critiques, and the skill itself stay in the **private** `superthinker` repo. Never put a PRD or critique in this repo.

## How a demo gets here

Superthinker does this automatically in Phase 7 (publish). For reference, the manual steps are:

```
gh api -X PUT repos/Copart-Prod-Design/superthinker-demos/contents/<slug>/index.html -f message="Add <slug> demo" -f content="$(base64 -i prototype.html)"
```

(macOS `base64` uses `-i <file>` and does not need `-w0`.)

## Repo setup (one-time — already done)

For the record, GitHub Pages on this repo is configured as:

- **`.nojekyll`** at the root — serve files statically, skip Jekyll (Jekyll errors on plain prototype HTML)
- **Pages source:** deploy from branch `main`, folder `/ (root)` (`build_type: legacy`, **not** GitHub Actions)

If a new demo's link 404s, check `gh api repos/Copart-Prod-Design/superthinker-demos/pages/builds/latest --jq '.status'` — it should say `built`.
